# 《等雨停》交互需求稿

## 1. 交互设计原则

### 1.1 场景化触发

本体验不使用显性的功能按钮作为主要交互入口。所有核心互动都通过场景中的物件、角色或环境区域触发，例如坐垫、角色、茶杯、风铃、灯、旧相册、紫阳花、石板路。

UI 元件只用于辅助表达状态，不应抢夺场景本身的注意力。

### 1.2 低压力反馈

每次交互反馈应轻、短、可忽略，不制造任务感。用户不需要理解复杂规则，也不需要完成清单。系统只在交互完成后用天气变化、音效变化、轻提示和角色气泡告诉用户：场景正在朝天晴变化。

### 1.3 状态可感知

用户应始终能感知当前天气阶段和距离天晴的变化趋势，但不要把它做成强游戏化进度条。天气进度模块应更像“雨势状态”或“天空变化提示”，而不是任务条。

## 2. 信息架构

画面由三层组成：

1. 场景层：房间、窗外、角色、可点击物件，是主要交互入口。
2. 反馈层：天气进度、天气变化 tips、角色气泡、物件动效。
3. 输入层：仅在触发聊天后出现的输入框。

## 2.1 全局交互状态

体验至少包含以下全局状态，所有 UI 和场景物件都应读取这些状态：

| 状态 | 默认值 | 影响范围 | 说明 |
|---|---|---|---|
| weatherTime | 0 | 天气阶段、进度条、背景、音效 | 自然计时与互动加速共同推进 |
| weatherStage | thunder | 天气 icon、背景图、环境音、tips | 根据 weatherTime 自动派生 |
| postureState | standing | 角色、茶杯、输入框、坐垫 | standing=未坐下，sitting=坐下时 |
| hasTea | false | 输入框、角色台词、茶杯完成态 | 喝茶后解锁更深入表达 |
| chatUnlocked | false | 输入框 | 喝茶后或特定角色互动后解锁 |
| endingReached | false | 天气 tips、路面、角色气泡 | 天晴后不再展示“朝晴天变化”的反馈 |

### 2.1.1 未坐下 / 坐下时

| 状态 | 场景表现 | 可用互动 | 反馈倾向 |
|---|---|---|---|
| 未坐下 | 右侧空地板为空，角色独自看雨 | 空地板、角色、灯、风铃、旧照片、紫阳花、路面提示 | 反馈更克制，角色少说话 |
| 坐下时 | 右侧出现用户剪影/陪坐位置，角色与用户共同看雨 | 茶杯、角色倒茶、聊天入口、路面走出去 | 反馈更温和，允许角色有更多回应 |

坐下不是通关前置条件。用户不坐下也能通过自然等待进入天晴，但坐下会解锁更完整的陪伴路径。

## 3. UI 元件清单

### 3.1 天气晴雨进度模块

#### 用途

展示当前天气阶段，以及场景正在从雨夜走向天晴的趋势。

#### 展示位置

建议放在画面顶部或窗边上方，轻量悬浮，不遮挡主要场景。

#### 表现形式

建议使用“天气阶段轨迹”而不是传统百分比进度条。

示例：

```text
雷雨中  ->  小雨  ->  雨停  ->  天晴
```

或：

```text
雨势：雷雨交加
天空仍很低，但雨声正在变轻
```

#### 状态

| 状态 | 计时范围 | 文案建议 | 视觉表现 |
|---|---:|---|---|
| 雷雨中 | 0-60 秒 | 雷雨中 | 深蓝紫、雨滴密集、闪电小图标 |
| 小雨 | 60-180 秒 | 小雨渐缓 | 蓝灰色、雨滴减少 |
| 雨停 | 180-300 秒 | 雨停了 | 浅蓝、云层变薄 |
| 天晴 | 300 秒后 | 天晴了 | 暖白/浅金、太阳或彩虹小图标 |

#### 交互规则

- 该模块不可点击。
- 当计时器因自然流逝或互动增加而跨阶段时，模块播放轻微过渡动效。
- 过渡动效应与天气 tips 同步出现。
- 当前阶段 icon 和已推进进度段需要有蓝紫色外发光，但发光范围克制。
- 进度线只连接天气 icon，不穿过 icon；进度填充展示当前实际计时进度。
- 模块底板与天气 tips 使用一致的玻璃态透明度、描边和模糊参数，避免天气进度模块显得比 tips 更重。

### 3.2 天气变化 Tips 元件

#### 用途

每次互动完成后，提示用户这次回应让天气朝天晴方向变化。

#### 展示位置

建议出现在画面右上方、天气进度模块下方，或靠近被互动的物件旁边。

#### 形式

轻 toast，不需要用户关闭，2-3 秒后自动消失。

当前 demo 规格：

- Toast 固定在天气进度模块下方。
- 宽度与天气进度模块一致。
- 左侧展示天气小 icon 和天气变化文案。
- 右侧依次展示“放晴值 +N”和消失倒计时。
- 透明度、描边、模糊与天气进度模块保持一致。

#### 文案原则

- 不写“进度 +10”这种系统化语言。
- 用情绪化但克制的表达。
- 让用户感到自己做的小动作有意义。

#### 文案示例

| 触发行为 | Tips 文案 |
|---|---|
| 坐下 | 雨声好像轻了一点。 |
| 角色倒茶 | 屋里的温度暖了一些。 |
| 喝茶 | 雨势正在慢慢缓下来。 |
| 摇风铃 | 风铃声盖过了一点风声。 |
| 调暗灯 | 房间安静了一点。 |
| 合上旧照片 | 那份沉默被好好放下了。 |
| 看紫阳花 | 雨里也有一点亮色。 |
| 走出去 | 云层散开了。 |

#### 状态

- 默认隐藏。
- 互动完成后出现。
- 如果短时间连续触发多个互动，tips 队列最多保留 1 条，后触发覆盖前一条。
- 天气阶段切换时，优先展示阶段切换 tips。

阶段切换 tips：

| 阶段 | 文案 |
|---|---|
| 进入小雨 | 大雨转小了。 |
| 进入雨停 | 雨停了，只剩下水滴声。 |
| 进入天晴 | 天晴了。 |

### 3.3 角色对话气泡元件

#### 用途

展示角色的短句回应。气泡是角色表达的主要 UI，但不应频繁打断用户。

#### 展示位置

靠近角色头部或身体上方，跟随角色位置。移动端可显示在角色附近的底部区域。

#### 视觉要求

- 半透明浅色气泡。
- 圆角克制，不做夸张漫画效果。
- 字数较短，最多 2 行。
- 气泡出现和消失都要柔和。

#### 状态

| 状态 | 说明 |
|---|---|
| 隐藏 | 默认状态，无对话时不显示。 |
| 出现中 | 角色被点击、物件触发或阶段切换时出现。 |
| 常驻短停留 | 重要台词停留 4-6 秒。 |
| 被新台词替换 | 新交互触发时直接替换气泡内容。 |
| 进入聊天 | 气泡下方出现输入框入口。 |

#### 与坐下状态关系

| 用户状态 | 角色反馈 |
|---|---|
| 未坐下 | 角色以轻动作回应为主，例如回头、停顿、看向窗外，不主动展开长对话 |
| 坐下时 | 角色可以倒茶、轻声回应，聊天入口可以被解锁 |
| 喝过茶 | 角色允许用户输入，不催促，不连续追问 |

#### 文案规则

- 台词短。
- 不追问。
- 不替用户解释情绪。
- 不把用户熬夜说成问题。
- 不猜测老照片含义。

示例：

- “今天也来得很晚。灯我调暗了。”
- “累的话，坐着一起放空也不错。”
- “那张照片，我夹在书里了。你没有说，我就没有问。”
- “雨太大了，不用着急出去。”
- “看，天晴了，没有不会结束的风雨。”

### 3.4 聊天输入框元件

#### 用途

用户主动触发后，提供与角色对话的入口。

#### 触发条件

只有在以下条件满足后出现：

1. 用户已经点击空地板坐下。
2. 用户已经点击茶杯喝茶，或点击角色进入更近距离互动。
3. 角色气泡提示“如果想说，我听着。”

#### 展示位置

建议从底部轻轻浮出，不遮挡角色和窗外天气。

#### 默认占位文案

- “如果想说，写一句就好。”
- “不用完整，几个字也可以。”

#### 组件状态

| 状态 | 说明 |
|---|---|
| 未解锁 | 默认不显示。 |
| 可唤起 | 喝茶后，角色气泡旁出现轻输入提示。 |
| 输入中 | 用户点击输入区后展开。 |
| 发送中 | 输入后等待 AI 回复。 |
| 已发送 | 角色气泡显示回复，输入框回到轻量状态。 |
| 收起 | 用户点击场景空白处，输入框收起。 |

#### 交互规则

- 不自动弹出键盘。
- 不强制用户输入。
- 用户不输入时，天气仍继续推进。
- 输入框不展示“任务完成”“亲密度增加”等强反馈。
- 用户发送内容后，计时器可增加 10 秒/轮。
- 输入框底板与天气 tips 透明度保持一致，不应比天气 tips 更厚重。
- 未解锁时可显示锁头图标和说明文案，锁头需使用图标而非文字字符。

### 3.5 场景热点提示元件

#### 用途

让用户知道哪些场景物件可互动，但不能做成明显按钮。

#### 表现方式

- 当前可交互物件应出现微弱的环境化特效提示，例如闪烁光粒子、轻微光圈、局部高光或极低透明度的呼吸光。
- 特效应不定时浮现和消失，像环境中的细小反光，而不是持续显示的按钮描边。
- 鼠标 hover 或移动端轻触时，物件提示可略微增强。
- 首次进入场景 3 秒后，可用一次性微光扫过关键物件，帮助用户理解“场景物件可以被触碰”。
- 特效需要服从当前天气阶段：雷雨阶段更冷、更弱；天晴阶段更暖、更清晰。

#### 视觉规格建议

| 提示类型 | 使用场景 | 表现 |
|---|---|---|
| 闪烁光粒子 | 小物件，如茶杯、旧相册、风铃 | 1-3 个小光点在物件附近短暂浮现，0.8-1.6 秒后消失 |
| 微弱光圈 | 中大型区域，如坐垫、石板路、紫阳花 | 柔和光圈从物件中心轻轻扩散，透明度低，不形成按钮感 |
| 局部高光 | 灯、杯沿、书页、风铃金属边缘 | 高光沿物件边缘短暂滑过 |
| 呼吸光 | 当前最推荐交互物件 | 低频明暗变化，周期 3-5 秒 |

#### 出现规则

- 同屏最多 3 个物件同时出现提示，避免画面变吵。
- 提示优先级：当前阶段关键物件 > 未完成物件 > 普通可重复物件。
- 同一物件提示间隔建议 4-8 秒，避免持续闪烁造成催促感。
- 用户完成某物件交互后，该物件提示减弱或消失。
- 阶段锁定物件可以有更弱提示，但点击后由角色气泡解释当前不可用原因。

#### 可交互物件

- 空地板 / 坐垫
- 角色
- 茶杯
- 风铃
- 灯
- 旧相册 / 书
- 紫阳花
- 石板路

#### 状态

| 状态 | 说明 |
|---|---|
| 可交互 | 间歇出现微弱光粒子或光圈，hover/轻触时提示增强。 |
| 推荐交互 | 除普通提示外，增加更明显但仍克制的呼吸光。 |
| 条件未满足 | 提示更弱，点击后无强响应，必要时角色轻声解释。 |
| 已完成 | 物件进入完成态，例如茶杯变空、灯光变暗、照片夹回书里，提示减弱或消失。 |
| 阶段锁定 | 例如 180 秒前石板路不可进入户外，仅保留极弱环境提示。 |

## 4. 场景物件交互状态

### 4.1 空地板 / 坐垫

#### 状态

| 状态 | 表现 | 点击结果 |
|---|---|---|
| 未坐下 | 右侧坐垫/空地板空着，有轻微可点击高光 | 用户坐下，计时器 +10 秒，右侧淡入用户剪影/陪坐状态 |
| 已坐下 | 坐垫出现用户剪影或视角变化 | 再次点击无新增计时，可轻微视角晃动 |

#### 反馈

- Tips：“雨声好像轻了一点。”
- 角色不一定说话，保持留白。

### 4.2 角色

#### 状态

| 状态 | 点击结果 | 角色反馈 |
|---|---|---|
| 用户未坐下 | 角色回头看镜头，计时器 +1 秒 | 可无台词，或“你来了。” |
| 用户已坐下 | 角色倒茶或轻微转身靠近，计时器 +1 秒 | “我倒了点热茶。” |
| 用户喝过茶 | 触发聊天入口，计时器 +10 秒/轮 | “如果想说，我听着。” |

#### 注意

角色不应主动连续发言。每次回应后进入安静状态。

### 4.3 茶杯

#### 状态

| 状态 | 点击结果 | 反馈 |
|---|---|---|
| 未坐下 | 无响应，计时器 +0 秒 | 可轻微杯光，无台词 |
| 已坐下未喝茶 | 用户喝茶，计时器 +5 秒 | 角色：“累的话，坐着一起放空也不错。” |
| 已喝茶 | 杯中茶减少，点击可听见杯声 | 不重复加计时 |

### 4.4 风铃

#### 状态

| 状态 | 点击结果 | 反馈 |
|---|---|---|
| 默认 | 风铃摇晃，计时器 +1 秒 | 风铃声与雨声相映 |
| 连续点击 | 只播放轻微摇晃 | 计时器不重复快速累加，需冷却 |

#### 冷却建议

同一物件 5 秒内只计一次有效互动。

### 4.5 灯

#### 状态

| 状态 | 点击结果 | 反馈 |
|---|---|---|
| 未调暗 | 灯光变柔 | 角色：“我把灯调暗了，可能会好睡一点。” |
| 已调暗 | 灯保持柔光 | 不重复触发台词 |

#### 设计含义

对应“用户最近经常熬夜”，但不责备用户。

### 4.6 旧相册 / 书 / 老照片

#### 状态

| 状态 | 点击结果 | 反馈 |
|---|---|---|
| 未发现 | 书页轻微翻动 | 角色：“那张照片，我夹在书里了。你没有说，我就没有问。” |
| 已打开 | 显示半遮挡照片，不展示清晰人物 | 出现轻选择：看一会儿 / 先合上 / 写一句 |
| 先合上 | 合上书，计时器 +5 秒 | “好。它不用今晚被解释。” |
| 写一句 | 展开输入框 | 占位：“不用完整，几个字也可以。” |

#### 注意

- 不识别照片人物。
- 不猜测照片关系。
- 不强制用户说明。

### 4.7 紫阳花

#### 状态

| 状态 | 点击结果 | 反馈 |
|---|---|---|
| 默认 | 花轻轻晃动，计时器 +1 秒 | 角色：“它们喜欢雨天。” |
| 天晴后 | 花颜色更亮 | Tips：“雨里也有一点亮色。” |

### 4.8 石板路 / 户外

#### 状态

| 状态 | 点击结果 | 反馈 |
|---|---|---|
| <180 秒 | 不进入户外 | 角色：“雨太大了，不用着急出去。” |
| >=180 秒 | 用户走出去，计时器直接到 300 秒 | 云散天晴、彩虹出现 |
| 已天晴 | 可看户外，不再推进 | 角色：“你改变了这场雨。” |

### 4.9 热点布点需求

当前 demo 需要至少覆盖以下互动点：

| 热点 | 建议位置 | 说明 |
|---|---|---|
| 风铃 | 画面上方风铃主体附近 | 点击触发风铃摇晃与声音变化 |
| 角色 | 角色背部/肩部附近 | 点击触发回头、倒茶或聊天入口 |
| 茶杯 | 茶具托盘附近 | 坐下后喝茶并解锁聊天 |
| 灯 | 左侧桌面灯附近 | 点击调暗灯光 |
| 旧相册/照片 | 左侧桌面或书架附近 | 触发老照片记忆 |
| 紫阳花 | 右侧窗外花丛附近 | 触发紫阳花台词 |
| 石板路 | 窗外中部路面 | 180 秒前提示等待，180 秒后走出去 |
| 空地板/坐垫 | 右下空木地板或坐垫附近 | 点击坐下，进入 sitting 状态 |

## 5. 天气阶段状态需求

### 5.1 阶段一：0-60 秒

#### 场景状态

- 色调灰暗压抑。
- 雷雨交加。
- 乌云密布。
- 大风。
- 风铃声音杂乱。

#### 可交互重点

- 坐下。
- 看角色。
- 调暗灯。
- 摇风铃。

#### 体验目标

让用户感受到压抑，但房间是安全的。

### 5.2 阶段二：60-180 秒

#### 场景状态

- 阴沉低压。
- 大雨转小雨。
- 雨声变细碎。
- 风声变规律。

#### 可交互重点

- 倒茶。
- 喝茶。
- 旧照片。
- 紫阳花。

#### 体验目标

角色开始用更明确的动作回应用户，但仍保持克制。

### 5.3 阶段三：180-300 秒

#### 场景状态

- 雨停。
- 云层仍在。
- 水滴声明显。
- 画面微亮。

#### 可交互重点

- 石板路变为可走出状态。
- 用户可以选择继续坐着，也可以走出去。

#### 体验目标

让用户获得主动权：继续待着也可以，走出去也可以。

### 5.4 阶段四：300 秒后

#### 场景状态

- 阳光投射。
- 蓝天白云。
- 花园彩虹。
- 风铃清脆。
- 有鸟鸣。

#### 可交互重点

- 角色总结短句。
- 场景可停留观看。

#### 体验目标

完成从压抑到舒展的情绪释放。

## 6. 全局状态变量

| weatherTime | number | 天气计时器，决定天气阶段 |
| weatherStage | enum | thunder / lightRain / cloudy / sunny |
| hasSatDown | boolean | 用户是否已坐下 |
| hasTea | boolean | 用户是否已喝茶 |
| hasDimmedLight | boolean | 是否已调暗灯 |
| hasTouchedWindBell | boolean | 是否已触发风铃 |

| isEnding | boolean | 是否已进入天晴结局 |

## 7. 反馈优先级

当多个反馈同时触发时，展示优先级如下：

1. 阶段切换反馈。
2. 角色关键台词。
3. 天气变化 tips。
4. 物件动效。
5. 普通音效。

示例：

用户喝茶后 weatherTime 从 58 秒变为 63 秒，同时触发喝茶反馈和阶段切换。此时优先展示“大雨转小了”，角色台词可延后 1 秒出现。

## 8. 异常与边界

- 用户不进行任何互动时，场景仍会自然推进到天晴。
- 用户快速连续点击同一物件时，不应无限叠加计时。
- 输入框展开时，点击场景空白处应收起输入框。
- 旧照片相关输入不得引导用户解释隐私。
- 180 秒前点击石板路不能硬性报错，只由角色温和提醒。
- 天晴后不再显示“天气朝晴天变化”的 tips。
