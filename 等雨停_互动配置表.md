# 《等雨停》互动配置表

本文档用于将 PRD 和交互需求稿中的规则转成可开发的配置。后续实现 HTML/CSS/JS demo 时，建议优先按本表搭建状态机和互动点点击逻辑。

## 1. 全局状态配置

## 2. 天气阶段配置

| stageId   |    时间范围 | 名称  | 色调   | 天气表现         | 音效             | 阶段切换文案      |
| --------- | ------: | --- | ---- | ------------ | -------------- | ----------- |
| thunder   |    0-60 | 雷雨中 | 灰暗压抑 | 雷雨交加、乌云密布、大风 | 沉闷雷声、急促风雨、风铃乱颤 | 雨声好像轻了一点。   |
| lightRain |  60-180 | 小雨  | 阴沉低压 | 大雨转小雨，小雨淅沥   | 细碎雨声、规律风声      | 大雨转小了。      |
| cloudy    | 180-300 | 阴天  | 平静微亮 | 雨停转阴天，空气潮湿   | 水滴声、轻微风声       | 雨停了，只剩下水滴声。 |
| sunny     |    300+ | 晴天  | 晴朗明媚 | 云散天晴、悬挂彩虹    | 温柔风声、清脆风铃、鸟鸣   | 天晴了。        |

阶段派生规则：

```js
function getWeatherStage(weatherTime) {
  if (weatherTime >= 300) return "sunny";
  if (weatherTime >= 180) return "cloudy";
  if (weatherTime >= 60) return "lightRain";
  return "thunder";
}
```

## 3.1 视频背景配置

| state         | standing 视频               | sitting 视频             | 播放规则                                     |
| ------------- | ------------------------- | ---------------------- | ---------------------------------------- |
| thunder       | `video_bg/thunder_default.mp4` | `video_bg/thunder_sit.mp4` | 移动端稳定版 H.264/AAC，单视频循环播放，切换时 3 秒叠化             |
| lightRain     | `video_bg/rainy_default.mp4`  | `video_bg/rainy_sit.mp4`  | 移动端稳定版 H.264/AAC，单视频循环播放，切换时 3 秒叠化             |
| cloudy        | `video_bg/cloud_default.mp4`  | `video_bg/cloud_sit.mp4`  | 移动端稳定版 H.264/AAC，单视频循环播放，切换时 3 秒叠化             |
| sunny         | `video_bg/sunny_default.mp4`  | `video_bg/sunny_sit.mp4`  | 移动端稳定版 H.264/AAC，自然进入天晴时按坐下状态选择对应循环视频         |
| roadEnding    | `video_bg/sunny_walk.mp4`     | 不区分坐下状态             | `weatherTime >= 180` 后点击路面触发，播放一次，不循环              |
| roadSunnyLoop | `video_bg/sunny_loop.mp4`     | 不区分坐下状态             | `roadEnding` 播放结束后切入并持续循环，不再切换 `sunny_sit.mp4` |

背景规则：

- 所有普通天气状态视频都循环播放，直到切换到下一个视频。
- 所有视频切换统一使用 3 秒叠化。
- 打开 HTML 时先黑屏，中间展示“进入”按钮；点击后点亮画面、开始播放当前视频并打开声音，同时 UI 淡入、天气计时开始。
- 天晴最终对白结束后，所有 UI 一同淡出，仅保留当前晴天视频画面。

| roadExit      | 石板路    |      是 | weatherTime >= 180 && endingReached=false   | 用户走出去，直接推进到天晴  | setTo300 | 云层散开了。      | 你改变了这场雨。                | weatherTime=300; endingReached=true |  0 |

说明：

- `addTime=setTo300` 表示不做加法，而是直接将 `weatherTime` 设置为 `300`。
- 同一物件在冷却时间内重复点击，只播放轻微物件反馈，不重复增加计时。
- 如果一次互动导致跨越天气阶段，优先展示阶段切换 toast，再展示角色气泡。
- 场景中最多同时显示 3 个互动点。默认初始显示 `character / tea / sitArea`。
- 每次有效点击后，该互动点立即隐藏并从 `visiblePoints` 移除；5 秒后再判断当前显示互动点是否少于 3 个。
- 如果 5 秒后少于 3 个，则从未显示、未冷却、未永久隐藏的候选池中随机浮现 1 个新互动点；即使场上远少于 3 个，每次点击也只补 1 个。
- 新浮现互动点执行 2 秒淡入动画。
- 被点击过的互动点进入 30 秒临时隐藏期；30 秒后回到候选池，等待后续补位判定时再次浮现。
- `sitArea` 是不可逆互动点：坐下后永久隐藏，不进入 30 秒恢复流程。

## 7. 推荐实现结构

```js
const state = {
  weatherTime: 0,
  postureState: "standing",
  hasTea: false,
  chatUnlocked: false,
  endingReached: false,
};

const hotspots = [
  {
    id: "sitArea",
    when: (s) => s.postureState === "standing",
    addTime: 10,
    toast: "雨声好像轻了一点。",
    onApply: (s) => {
      s.postureState = "sitting";
    },
  },
  {
    id: "teaDrink",
    when: (s) => s.postureState === "sitting" && !s.hasTea,
    addTime: 5,
    toast: "雨势正在慢慢缓下来。",
    bubble: "累的话，坐着一起放空也不错。",
    onApply: (s) => {
      s.hasTea = true;
      s.chatUnlocked = true;
    },
  },
];
```

## 8. 验收检查点

| 检查项   | 通过标准                            |
| ----- | ------------------------------- |
| 自然推进  | 不点击任何物件时，天气仍会从雷雨推进到天晴           |
| 坐下状态  | 点击空地板后进入 sitting 状态，茶杯和聊天路径变得可用 |
| 天气加速  | 有效互动后 weatherTime 正确增加，并刷新天气预测  |
| 路面限制  | 180 秒前点击路面只提示，不进入天晴             |
| 终局    | 180 秒后点击路面直接进入 300 秒天晴阶段        |
| UI 反馈 | toast、放晴值、倒计时、角色气泡不会互相遮挡        |
| 情绪语气  | 角色不追问、不分析、不替用户解释照片              |
